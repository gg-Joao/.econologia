## CASO DE USO: Entrar no Sistema

Atores envolvidos: Usuário (Morador, Cooperativa ou Administrador)

Pré-condições: O usuário possui cadastro no sistema.

Pós-condições: O usuário é autenticado e tem acesso ao sistema.

Fluxo principal:
 1. O usuário acessa a opção Entrar no Sistema.

 2. O usuário informa login e senha.

 3. O sistema valida as credenciais.
 
 4. O sistema libera o acesso ao sistema.

Fluxo alternativo: Senha incorreta
O sistema informa erro e solicita nova tentativa.

Fluxo de exceção: Conta inexistente
O sistema informa que a conta não existe e encerra o processo.

Modelo de interação associado: Diagrama de Comunicação.
